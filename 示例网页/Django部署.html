Django部署

@(Django)[Apache Http Server|mod_wsgi|Nginx|fastcgi]
针对Windows平台Django的部署，所使用的数据库为MS SQL 2008+

环境说明

  1. 操作系统：window7+
  2. Python： python3.3.5(32位版)
  3. Django： Django 1.6.5版 可以通过pip install Django==1.6.5安装
  4. Apache Http Server： 2.2.25版本
  5. Mod_wsgi：下载地址 需对应Apache Http Server版本和Python版本 (此处对应为mod_wsgi-3.5.ap22.win32-py3.3.zip即Apache2.2及Python3.3)
  6. Nginx： 1.6.1版本
  7. Fastcgi： flup-py3(针对Python3版本) 需要通过python setup.py install来安装
  8. 针对MS SQL Server数据库的Python组件：
    • 针对MS SQL SERVER 数据库，推荐使用的组件为django-mssql 该组件支持 python3.x ，在安装该组件时需要注意：
    • 针对于Python3.x版本的django-mssql需要去Bitbucket下载，再通过python setup.py install 来安装，pypi上版本过旧，不能通过pip install 来安装。
    • 该组件以来PyWin32，下载地址自行选择对应版本下载。
    • django-mssql说明文档。可参考文档进行操作。

以下两个组件也可以实现MS SQL SERVER的访问，但现有版本在部署到Apache服务器上时，会出现错误
* Windows ODBC 支持： pyodbc (3.0.7) win32 for py3.3 注意版本对应
* Django ODBC 支持： django-pyodbc-azure (1.1.5) 可直接通过pip install django-pyodbc-azure安装

注意：

若使用Apache Http Server作为服务器，则6与7不需要安装 推荐
若使用Nginx作为服务器，则4与5不需要安装

注意：部署Django之前先要修改Django的setting.py文件中的设置，如下

    DEBUG = False
    TEMPLATE_DEBUG = DEBUG
    ALLOWED_HOSTS = ['*']

Apache Http Server + Mod_wsgi

参考自CSND博客 Windows下配置Django 1.6+apache+mod_wsgi

1. 修改Django自动生成的wsgi.py文件

"""
WSGI config for myweb project.

It exposes the WSGI callable as a module-level variable named ``application``.

For more information on this file, see
https://docs.djangoproject.com/en/dev/howto/deployment/wsgi/
"""

import os,sys
os.environ.setdefault("DJANGO_SETTINGS_MODULE", "dgongdvps.settings")
sys.path.append(r'D:/ProgEnviron/servers/dgongdvps/') #必须自己加上
from django.core.wsgi import get_wsgi_application
application = get_wsgi_application()

2. 安装Apache Http Server和Mod_wsgi

  • 安装好Apache服务器
  • 下载Mod_wsgi,解压后将mod_wsgi.so放入Apache安装目录下的modules目录下

3. 配置Apache Http Server和Mod_wsgi

  • 打开Apache安装目录的conf文件夹，打开httpd.conf文件
  • 找到LoadModule那一段，添加：
    LoadModule wsgi_module modules/mod_wsgi.so
    WSGIScriptAlias / "D:/ProgEnviron/servers/dgongdvps/dgongdvps/wsgi.py"
  • 找到Directory的配置区域，添加如下的一段配置：
    <Directory "D:/ProgEnviron/servers/dgongdvps/">
        Order allow,deny
        Allow from all
    </Directory>
  • 重启apache服务
    注意：
    • 若在重启apache报出Internal Server Error错误，那么可能是Django设置文件未配置。
    • 若报出the requested operation has failed的错误，先排查80端口是否被占用。
    • 若80端口未被占用，打开apache安装目录的bin文件夹，输入httpd –t的命令查看httpd.conf文件中那里出错

4. 若要优化服务器，还需要更多其他配置

Nginx + fastcgi

1. 安装Nginx以及flup-py3

  • Nginx下载Windows版本，解压即可
  • flup-py3 需要通过python setup.py install来安装

2. 配置Nginx，Nginx目录下conf文件中nginx.conf文件

http {
    include       mime.types;
    default_type  application/octet-stream;

    sendfile        on;
    #keepalive_timeout  0;
    keepalive_timeout  65;

    #gzip  on;

    server {
        listen       80;
        server_name  localhost;

        #host and port to fastcgi server
        location / {
            fastcgi_pass 127.0.0.1:8080;#与Django服务的地址与端口相同
            fastcgi_param PATH_INFO $fastcgi_script_name;
            fastcgi_param REQUEST_METHOD $request_method;
            fastcgi_param QUERY_STRING $query_string;
            fastcgi_param CONTENT_TYPE $content_type;
            fastcgi_param CONTENT_LENGTH $content_length;
            fastcgi_param pass_header Authorization;
            fastcgi_intercept_errors off;
        }
        #配置静态文件夹
        location ~* ^.+\.(gif|png|jpg|jpeg|css|swf|htm|html|asp|php|jsp|js|doc|txt)$ {
            root   D:/ProgEnviron/servers/dgongdvps/webapps/components;
            #access_log   off;
            expires 1d;#设置缓存时间为一天
        }

        #error_page  404              /404.html;

        # redirect server error pages to the static page /50x.html
        #
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }
    }

以上仅为修改部分配置，并非完整配置

3. 在工程目录下添加一个run.bat文件

文件内容为

    python manage.py runfcgi method=threaded host=<主机IP/域名> port=<端口>

说明：主机IP和端口需要和fastcgi_pass项匹配，以上面fastcgi_pass 127.0.0.1:8080;为例：
<主机IP/域名>：127.0.0.1
<端口>：8080
启动Nginx，再双击run.bat启动django服务
